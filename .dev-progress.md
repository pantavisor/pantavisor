## Design decisions
- Used existing `pv_platform_status_string()` for platform event names (uppercase: STARTED, READY, STOPPED) rather than pitch's lowercase, for consistency with existing code
- "wrapped" event emitted only on first overflow (when head wraps back to 0) as a simple signal
- No deduplication per pitch spec
